# 个人习惯、偏好和注意事项

这份文档不是正式工程规范，主要是记录我个人在和 ChatGPT / Codex 协作时的一些习惯、偏好和注意事项，后面可以持续往里补。

## 计算资源和执行方式

1. 对于计算量小的工作，可以直接在本地完成。
2. 对于计算量大的工作，我更希望 Codex 不要直接默认在本地长时间执行，而是把需要执行的命令整理写入 `docs/run_order.md`，让我转移到 Linux 服务器上执行。
3. 我会在 Linux 服务器上执行这些命令，再把执行结果、日志或报错反馈回来，供后续继续分析和推进。

## Prompt 和反馈的文档存储方式

1. 我希望将 prompt 和 Codex 执行完 prompt 之后的 feedback，按 stage 存储在 `docs/ai/` 下。
2. 后续如果要新建这类文档，优先按 stage 做清晰分类和命名，方便回看每个阶段当时是怎么想、怎么做、做完后的反馈是什么。
